# 每日營養需求計算器 

一個基於純前端技術（HTML5 / CSS3 / Vanilla JS）打造的精準營養與熱量消耗計算工具。無需任何後端伺服器或 API 串接，所有計算均在瀏覽器本地即時完成，兼具極佳的隱私性與回應速度。

> 💡 **科學依據與公式證明**：本工具底層的所有數學模型、動態係數設計以及完整的學術文獻來源，均詳細記載於 [proof.md](./proof.md) 文件中。

## 🌟 核心獨特性與設計優勢

相較於網路上常見的簡易 TDEE 計算器，本專案在計算邏輯與架構設計上具備以下顯著優勢（詳細數學推導請參閱 [演算法證明文件](./proof.md)）：

1. **動態時數與多項運動疊加（ACSM METs 標準）**
   * **一般計算器：** 通常僅能選擇模糊的活動量區間（如：輕度、中度、重度），無法精確反映當天的真實活動。
   * **本計算器：** 採用美國運動醫學會（ACSM）的代謝當量（MET）標準，支援**動態新增多項運動與勞動類型**。系統會自動根據每項運動的時間與使用者的體重，精確計算出該項運動的淨消耗熱量 ($E_{ex}$)。

2. **精細化的時間與熱量拆解模型**
   * 為了避免運動時數與日常靜態消耗重複計算，本系統引入了 24 小時時間權重拆解公式：
     * **睡眠消耗 ($E_{sleep}$)**：固定以基礎代謝率 (BMR) 的 $0.95$ 倍進行 8 小時基礎估算。
     * **靜態清醒 ($E_{rest}$)**：扣除睡眠與運動時數後的剩餘時間，以完整 BMR 權重計算。
     * **運動消耗 ($E_{ex}$)**：依據實際運動類型與時數加權。
     * 最終 TDEE 由此三者嚴謹相加，大幅降低傳統公式直接乘上活動係數帶來的誤差。

3. **動態蛋白質與巨量營養素配比**
   * 蛋白質攝取係數 ($f_{prot}$) 會根據當天總運動時數動態調整（從無運動的 $0.8\,\text{g/kg}$，到高強度運動下的 $2.0\,\text{g/kg}$），確保增肌或減脂族群能獲得更科學的巨量營養素（碳水、蛋白、脂肪）建議。

4. **全面性的微量元素引導（DRI 標準）**
   * 整合世界衛生組織 (WHO) 與膳食營養素參考攝取量 (DRI)，根據使用者輸入的**性別**與**年齡**，自動切換並輸出 11 種維生素與 10 種礦物質的每日建議量（例如：針對不同年齡層自動調整維生素 D、鈣、鐵的防護標準）。

## 📅 適用情境

* **健身與體重管理者**：需要精確紀錄當天有氧 or 重訓後的真實 TDEE 變化，進而調配飲食者。
* **飲食精算族群**：不滿足於粗估數據，希望根據每日實際作息（如辦公靜坐 vs 戶外快走）動態調整攝取量者。
* **健康教育與自主檢視**：想了解自身性別與年齡層對應的微量元素（鈣、鋅、鎂、維生素群）基礎需求的長者或一般大眾。

## 🛠️ 技術特點

* **零外部依賴**：純原生 JavaScript、CSS 變數與 Flexbox/Grid 佈局，無引入任何龐大的前端框架或第三方套件，網頁載入流暢。
* **高隱私性**：所有身體數據與運動紀錄皆留在使用者瀏覽器中，絕不上傳至任何外部伺服器。
* **響應式設計 (RWD)**：完美適配手機、平板與桌上型電腦螢幕，方便隨手開啟記錄。
* **嚴謹學術實作**：核心計算非憑空捏造，公式代碼嚴格遵循臨床文獻與官方指引，完整佐證請見 [proof.md](./proof.md)。
